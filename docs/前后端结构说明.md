## 后端目录结构

```text
backend/
  app.py              # API 入口
  settings.py         # 配置
  bootstrap.py        # import 路径初始化
  agent/              # Agent 主循环、工具注册、工具实现
  context/            # 上下文压缩、摘要、rolling state
  image/              # ComfyUI / 图片生成 / 质量评分
  llm/                # Ollama/Qwen、HF、本地模型路由
  retrieval/          # RAG 知识库、LinUCB/DDQN/RL 检索策略
  storage/            # 数据库、会话、训练记录
  workspace/          # 本地文件读写、搜索、沙盒权限
```

### 根目录文件

- `backend/app.py`：后端 API 业务入口。前端请求会先进入这里，再分发到聊天、知识库、训练、图片、地图、文件等模块。
- `backend/settings.py`：统一配置文件。管理 Ollama/Qwen、HF、本地路径、RAG、ComfyUI、数据库、工作区等环境变量。
- `backend/bootstrap.py`：脚本启动时补齐 Python import 路径，保证 `scripts/` 下的训练和评测脚本能正确导入后端模块。

### 子包说明

- `backend/agent/`：Agent 主循环、工具注册和工具实现。负责让 Qwen 判断是否检索知识库、读文件、生成图片或调用其他工具。
- `backend/context/`：上下文压缩模块。负责把长对话压成摘要、维护 rolling state，并在新一轮对话时重新装配相关上下文。
- `backend/image/`：图片生成模块。负责 ComfyUI 调用、OpenAI 图片接口预留、图片保存、prompt 改写、候选图评分和重试。
- `backend/llm/`：模型调用模块。负责 Ollama/Qwen、HF 本地模型、模型状态检查和模型路由。
- `backend/retrieval/`：知识库和强化学习检索模块。负责 RAG 知识库构建、知识检索、LinUCB/DDQN/PPO/DPO/ORPO 检索策略环境。
- `backend/storage/`：数据存储模块。负责 MySQL、本地会话、聊天历史、训练记录、评测记录和轨迹实验记录。
- `backend/workspace/`：本地工作区工具模块。负责文件读取、搜索、索引、修改提案、写入校验和沙盒路径保护。

## 项目图片生成 Prompt 模板

这里记录的是本项目内部真正用于文生图的 Prompt 逻辑，不是外部 PPT 示意图提示词。核心代码位置如下：

```text
backend/agent/loop.py              # 判断用户是不是要生成图片，并调用 generate_image_advanced
backend/image/quality.py           # 改写 positive prompt / negative prompt，并做质量评分和重试
backend/image/comfyui_workflow.py  # 把 positive / negative prompt 填入 ComfyUI 工作流节点
```

### 1. Agent 调用图片生成工具时的输入

```python
tool_input = {
    "prompt": query,
    "quality_mode": "high",
    "batch_size": 1,
    "allow_retry": True,
    "use_highres_fix": True,
}
```

说明：用户在聊天框输入“生成、画、绘制、海报、logo、产品图”等图片生成请求时，Agent 会把原始问题作为 `prompt` 传给高级图片生成工具。

### 2. 通用图片正向 Prompt 模板

```text
{raw_prompt}. Clear primary subject, entire main subject visible unless a close-up is requested, coherent composition, realistic proportions, {final_style} style, sharp focus, balanced lighting, detailed surfaces, high quality, clean background where appropriate, no watermark.
```

说明：这是默认 `general_photorealistic` 模式，适合普通场景图。它会强制强调主体清晰、构图稳定、比例真实、光照均衡和高质量细节。

### 3. 产品图正向 Prompt 模板

```text
Professional studio product photography of {raw_prompt}. Entire product fully visible in frame, centered composition with generous margins, clean seamless background, realistic proportions, accurate material texture, softbox lighting, subtle shadow on the table, sharp focus, high detail, commercial catalog quality, no text, no watermark.
```

说明：当 prompt 命中 product、shoe、watch、phone、产品、商品等关键词时，会使用产品图模板，适合生成电商展示图、商品图、鞋类产品图。

### 4. 汽车图正向 Prompt 模板

```text
A single {raw_prompt}, one car only, parked alone or shown as the only main vehicle. Low-angle front three-quarter view, closed doors, intact body panels, realistic proportions, detailed headlights, realistic wheels, clean aerodynamic lines. Cinematic lighting, glossy reflections, high-end automotive photography, sharp focus, photorealistic, highly detailed, no people, no text, no watermark, no other vehicles.
```

说明：当 prompt 命中 car、supercar、vehicle、跑车、汽车、赛车等关键词时，会使用汽车专用模板，重点限制“单车、车身完整、轮胎真实、不要多余车辆”。

### 5. 通用负向 Prompt 模板

```text
low quality, blurry, distorted, deformed, bad anatomy, extra objects, duplicate objects, text, watermark, logo, signature, oversaturated, unrealistic lighting, bad perspective, cropped subject, floating parts
```

说明：这是所有图片都会拼接的基础负向 prompt，用来减少低质量、模糊、畸形、重复主体、水印、文字、错误透视等问题。

### 6. 不同预设追加的负向 Prompt

```text
automotive_photorealistic:
multiple cars, extra vehicles, duplicate car, background car, open doors, broken body, deformed body, malformed wheels, extra wheels, missing wheels, distorted perspective, bad reflections, mirrored vehicle, floating parts, disassembled car, unrealistic car structure, extra headlights, extra mirrors, people, text, watermark, logo

product_photography:
warped product, duplicate product, wrong label, fake text, messy background, broken geometry, inconsistent material, bad reflection

portrait_photorealistic:
extra fingers, missing fingers, malformed hands, deformed face, asymmetrical eyes, bad teeth, extra limbs, broken anatomy, distorted body

poster_design:
garbled text, watermark, signature, low quality, cluttered layout

logo_draft:
photorealistic, cluttered, complex background, watermark, signature, fake text

interior_design:
warped furniture, impossible geometry, clutter, low quality, text, watermark, distorted room perspective
```

说明：`ImageNegativePromptBuilder` 会把通用负向 prompt 和当前 preset 的负向 prompt 合并去重，形成最终传入 ComfyUI 的 `negative_prompt`。

### 7. 图片质量失败后的重试 Prompt

```python
positive = f"{plan.positive_prompt}, {', '.join(prompt_fixes)}"
negative = f"{plan.negative_prompt}, duplicate object, extra object, malformed structure, broken geometry"
```

说明：如果第一轮候选图评分不达标，系统会自动追加“单主体、居中构图、不要重复物体”等修复词，并把重复物体、结构错误、几何畸形加入负向 prompt。

### 8. 最终进入 ComfyUI 的位置

```python
"6": {"class_type": "CLIPTextEncode", "inputs": {"text": plan.positive_prompt, "clip": ["4", 1]}},
"7": {"class_type": "CLIPTextEncode", "inputs": {"text": plan.negative_prompt, "clip": ["4", 1]}},
```

说明：项目最终不是直接把用户原句送给 ComfyUI，而是先生成 `ImageGenerationPlan`，再把改写后的 `positive_prompt` 和 `negative_prompt` 分别填入 ComfyUI 的正向、负向 CLIP 编码节点。
